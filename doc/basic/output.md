# Output Part Specification

## Purpose
Define how results are delivered from the scaffolding workflow using the **Model–View–Control** structure.

## Model
- Store final assessment outcomes and decision metadata.
- Version outputs for audit and comparison.
- Maintain links to originating inputs and assessment evidence.

## View
- Present summarized decision outcomes and rationale.
- Provide detailed drill-down for metrics, assumptions, and risks.
- Export outputs in markdown, JSON, or report-friendly format.

## Control
- Aggregate processed results from assessment steps.
- Apply output rules (visibility, approval state, distribution).
- Trigger downstream notifications or integrations.
